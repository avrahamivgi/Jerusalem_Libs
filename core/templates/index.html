Library API Instructions

To use the API, you need to be signed in. If you haven't signed up yet, you can do so here.

After signup your personal library db system will be created.

This is the base path for the API: http://localhost:8000/api/v1/

You will use him for the api calls. like this: http://localhost:8000/api/v1/customers

Customer API

GET /customers: Retrieve all customers or a specific customer by ID.

(Opitional Query Parameter: id)

POST /customers: Add a new customer.

(Reqiuerd Body Customer data : name= , age= , customer_id=)

PUT /customers: Update the name or the age of a customer.

(Reqiuerd Query Parameter: id | Request Body:name= , age= )

DELETE /customers: Delete a customer if he don't have any associated rents.

(Reqiuerd Query Parameter: id)

Book API

GET /books: Retrieve all books or multiply books by name filtering.

(Opitional Query Parameter: name)

POST /books: Add a new book.

(Reqiuerd Body Book data : name= , author= , retrun_period=)

PUT /books: Update the return period of a book if he not currently in rent.

(Reqiuerd Query Parameter: id| Request Body: retrun_period)

DELETE /books: Delete a book if it doesn't have any associated rents.

(Reqiuerd Query Parameter: id)

Rent API

GET /rents: Retrieve all rents or rents by customer ID.

Add the query parameter lated=true to retrieve only late rents.

Add the query parameter id= to retrieve rents of specific customer.

(Opitional Query Parameter: id, lated)

POST /rents: Add a new rent if the requested book not rented.

(Reqiuerd Body Rent - id's of attirbutes: cust= , book=)

DELETE /rents: Delete a rent.

(Reqiuerd Query Parameter: id)

User API

POST /signup: Create a new user. (Request Body: User data)

GET /private: Access a private endpoint. Requires authentication.
